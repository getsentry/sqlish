## Long-term Knowledge

### Architecture

<!-- lore:019d1c89-c211-7d1f-85a5-e1a01ac6d60c -->

- **sqlish package structure and build pipeline**: Package \`@sentry/sqlish\` is a SQL-ish parser/formatter with two entry points: \`src/index.ts\` (core) and \`src/react.tsx\` (React formatter). Built with tsup (ESM, declarations, sourcemaps). PEG grammar is pre-compiled to \`src/sqlish-grammar.js\` via \`scripts/compile-grammar.mjs\` (peggy CLI) in the \`prebuild\` step — this file is gitignored. Zero runtime deps; React is an optional peer dep (>=17). Uses pnpm as package manager, vitest for tests, and Craft for releases via \`.craft.yml\` (npm public + github targets).

### Decision

<!-- lore:019d1c89-c21a-7648-ac34-20b6d44dddee -->

- **Removed Sentry instrumentation from sqlish library code**: All \`@sentry/react\` instrumentation (spans, captureException) was removed from \`SQLishFormatter\` when making the package publishable. The library is a dependency consumed by other Sentry projects — it should not depend on or call Sentry SDK directly. The formatter was also converted from \`.tsx\` to \`.ts\` since React rendering was moved to the separate \`/react\` subpath export.

### Pattern

<!-- lore:019d1c89-c216-74c4-bd17-054ac33ef099 -->

- **sqlish grammar import uses generated JS file**: \`SQLishParser.ts\` imports from \`./sqlish-grammar.js\` which is generated by \`scripts/compile-grammar.mjs\` during \`prebuild\`. A corresponding \`src/sqlish-grammar.d.ts\` provides types. The generated file is gitignored, so you must run \`pnpm run prebuild\` (or \`pnpm run build\` which triggers it) before TypeScript compilation or IDE type-checking will work.
