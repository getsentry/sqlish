# Implementation Plan: Publish `@sentry/sqlish` via Craft

## Summary

Transform the `sqlish` repo into a publishable `@sentry/sqlish` npm package with:
- ESM-only output via tsup
- React code as a separate `/react` subpath export (optional peer dep)
- All Sentry instrumentation removed
- Craft-based release pipeline with GitHub Actions CI

---

## Design Decisions (Confirmed)

| Decision | Choice |
|----------|--------|
| Sentry instrumentation | **Remove completely** — no @sentry/react, no @sentry/core |
| Output format | **ESM-only** |
| Build tool | **tsup** (esbuild-based) |
| React | **Optional peer dependency**, separate `/react` entry point |
| PEG grammar | **Pre-compile** via peggy CLI in a prebuild script |

---

## Phase 1: Remove Sentry Instrumentation & Restructure Source

### 1.1 Rewrite `src/SQLishFormatter.tsx` → `src/SQLishFormatter.ts`

**File: `src/SQLishFormatter.ts`** (rename from .tsx, remove all React/Sentry)

Remove:
- `import * as Sentry from '@sentry/react'`
- `Sentry.startInactiveSpan(...)` span creation
- `Sentry.withScope(...)` / `Sentry.captureException(...)` error reporting
- `toSimpleMarkup()` method (moves to react entry point)
- `Format.SIMPLE_MARKUP` enum variant and its formatter reference
- The `simpleMarkup` import

Keep:
- `toString()` method that calls `toFormat()` with string formatter
- Parser instantiation
- Error handling (catch parse errors, return original SQL string)

Result — a pure TS file with no JSX, no React, no Sentry:

```typescript
import type { Token } from './types';
import { string } from './formatters/string';
import { SQLishParser } from './SQLishParser';

type StringFormatterOptions = Parameters<typeof string>[1];

export class SQLishFormatter {
  parser: SQLishParser;

  constructor() {
    this.parser = new SQLishParser();
  }

  toString(sql: string, options?: StringFormatterOptions): string {
    let tokens: Token[];
    try {
      tokens = this.parser.parse(sql);
    } catch {
      return sql;
    }
    return string(tokens, options);
  }
}
```

### 1.2 Create React entry point

**New file: `src/react.tsx`**

```typescript
export { simpleMarkup } from './formatters/simpleMarkup';
```

This is the separate entry point for `@sentry/sqlish/react`. It re-exports the React formatter. Consumers who don't need React never import this.

### 1.3 Update barrel export

**File: `src/index.ts`** (rename from `src/sqlish.ts`)

```typescript
export { SQLishParser } from './SQLishParser';
export { SQLishFormatter } from './SQLishFormatter';
export { string } from './formatters/string';
export type { Token } from './types';
```

Changes:
- Rename `sqlish.ts` → `index.ts` (standard barrel name)
- Export `Token` type (was commented out)
- Export `string` formatter directly for consumers who want low-level access
- Do NOT export simpleMarkup here (it's in `/react`)

### 1.4 Clean up `src/formatters/simpleMarkup.tsx`

No code changes needed — it only imports `Token` type, uses JSX with no Sentry deps. It stays as-is.

### 1.5 Delete webpack artifacts

**Delete:**
- `webpack.config.js`

These are replaced by the tsup build + peggy prebuild script.

## Phase 2: Build System — tsup + PEG Pre-compilation

### 2.1 Pre-compile PEG grammar

**New file: `scripts/compile-grammar.mjs`**

```javascript
import { readFileSync, writeFileSync } from 'node:fs';
import peggy from 'peggy';

const grammar = readFileSync('src/sqlish.pegjs', 'utf8');
const source = peggy.generate(grammar, {
  output: 'source',
  format: 'es',       // ESM output
  optimize: 'speed',
  trace: false,
  cache: false,
});

writeFileSync(
  'src/sqlish-grammar.js',
  source,
);
```

This outputs `src/sqlish-grammar.js` (already in `.gitignore`). The generated file is an ESM module with a `parse` function export.

### 2.2 Update grammar import

**File: `src/SQLishParser.ts`** — change import:

```typescript
import type { Token } from './types';
// @ts-expect-error — generated file, no .d.ts
import { parse } from './sqlish-grammar.js';

export class SQLishParser {
  parse(sql: string): Token[] {
    return parse(sql) as Token[];
  }
}
```

### 2.3 Update type declaration for grammar

**File: `src/pegjs.d.ts`** — replace with declaration for the generated file:

```typescript
declare module './sqlish-grammar.js' {
  export function parse(input: string): unknown;
}
```

Or alternatively, delete `pegjs.d.ts` entirely and use the `@ts-expect-error` approach in 2.2.

**Recommended:** Keep a `.d.ts` for cleaner types, rename to `src/sqlish-grammar.d.ts`:

```typescript
export function parse(input: string): unknown;
```

### 2.4 Configure tsup

**New file: `tsup.config.ts`**

```typescript
import { defineConfig } from 'tsup';

export default defineConfig([
  {
    entry: {
      index: 'src/index.ts',
    },
    format: ['esm'],
    dts: true,
    outDir: 'dist',
    clean: true,
    sourcemap: true,
  },
  {
    entry: {
      react: 'src/react.tsx',
    },
    format: ['esm'],
    dts: true,
    outDir: 'dist',
    clean: false,       // Don't clean — first build already ran
    sourcemap: true,
    external: ['react', 'react-dom'],
    esbuildOptions(options) {
      options.jsx = 'automatic';
    },
  },
]);
```

This produces:
- `dist/index.js` + `dist/index.d.ts` (main entry)
- `dist/react.js` + `dist/react.d.ts` (react entry)

```yaml
name: CI
on:
  push:
    branches: [main]
  pull_request:

jobs:
  build-and-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - run: npm ci

      - run: npm run build

      - run: npm test

      # Upload build artifact for Craft releases
      - name: Pack tarball
        run: npm pack

      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: ${{ github.sha }}
          path: '*.tgz'
          if-no-files-found: error
```

The artifact name `${{ github.sha }}` is required by Craft to find the build artifact.

## Phase 5: Jest Configuration Update

### 5.1 Update `jest.config.js`

The grammar is now pre-compiled, so jest needs to:
1. Handle the `.js` grammar import
2. Support ESM or use ts-jest with proper module mapping

```javascript
/** @type {import('ts-jest').JestConfigWithTsJest} */
export default {
  preset: 'ts-jest/presets/default-esm',
  testEnvironment: 'node',
  roots: ['<rootDir>/src'],
  testMatch: ['**/__tests__/**/*.ts', '**/?(*.)+(spec|test).ts', '**/?(*.)+(spec|test).tsx'],
  moduleNameMapper: {
    '^(\\.{1,2}/.*)\\.js$': '$1',
  },
  transform: {
    '^.+\\.tsx?$': ['ts-jest', { useESM: true }],
  },
  collectCoverageFrom: [
    'src/**/*.{ts,tsx}',
    '!src/**/*.d.ts',
    '!src/**/*.spec.{ts,tsx}',
    '!src/**/*.test.{ts,tsx}',
  ],
};
```

Changes:
- ESM preset for ts-jest
- `moduleNameMapper` to resolve `.js` extensions back to source
- Remove `setupFilesAfterSetup` (file doesn't exist)
- Export as ESM (matches `"type": "module"`)

Note: Tests require running `npm run prebuild` first so the grammar .js file exists.

## Usage

### Parsing

```typescript
import { SQLishParser } from '@sentry/sqlish';

const parser = new SQLishParser();
const tokens = parser.parse('SELECT id, name FROM users WHERE id = 42');
```

### String Formatting

```typescript
import { SQLishFormatter } from '@sentry/sqlish';

const formatter = new SQLishFormatter();
const formatted = formatter.toString('SELECT id, name FROM users WHERE id = 42');

import { SQLishParser } from '@sentry/sqlish';
import { simpleMarkup } from '@sentry/sqlish/react';

const parser = new SQLishParser();
const tokens = parser.parse('SELECT id, name FROM users WHERE id = 42');
const highlighted = simpleMarkup(tokens);

return <div>{highlighted}</div>;
```
