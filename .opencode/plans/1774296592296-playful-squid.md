# Plan: Publish `@sentry/sqlish` via Craft

## Overview
Transform the `sqlish` repo into a publishable `@sentry/sqlish` npm package with Craft-based releases, React as an optional peer dep via a `/react` subpath export, and zero runtime dependencies. Uses pnpm + vitest + tsup.

---

## Phase 1: Restructure Source Code

### 1.1 Rewrite `src/SQLishFormatter.tsx` → `src/SQLishFormatter.ts`
- Remove `@sentry/react` import and all Sentry instrumentation (spans, captureException)
- Remove `toSimpleMarkup()` method, `Format` enum, `FORMATTERS` map, and `simpleMarkup` import
- Keep `toString()` with direct parsing + string formatting + error fallback
- Rename from `.tsx` to `.ts` (no more JSX)

Resulting file:
```typescript
import type { Token } from './types';
import { string } from './formatters/string';
import { SQLishParser } from './SQLishParser';
type StringFormatterOptions = Parameters<typeof string>[1];

export class SQLishFormatter {
  parser: SQLishParser;
  constructor() {
    this.parser = new SQLishParser();
  }
  toString(sql: string, options?: StringFormatterOptions): string {
    let tokens: Token[];
    try {
      tokens = this.parser.parse(sql);
    } catch {
      return sql;
    }
    return string(tokens, options);
  }
}
```

- Export `Token` type (was commented out)

## Phase 2: Build System — tsup + PEG Pre-compilation

### 2.1 Create `scripts/compile-grammar.mjs`
Pre-compiles PEG grammar to ESM JS (already in `.gitignore` as `src/sqlish-grammar.js`):
```javascript
import { readFileSync, writeFileSync } from 'node:fs';
import peggy from 'peggy';
const grammar = readFileSync('src/sqlish.pegjs', 'utf8');
const source = peggy.generate(grammar, {
  output: 'source', format: 'es', optimize: 'speed',
});
writeFileSync('src/sqlish-grammar.js', source);
```

### 2.2 Update `src/SQLishParser.ts`
Change import from `.pegjs` module to generated `.js` file:
```typescript
import type { Token } from './types';
import { parse } from './sqlish-grammar.js';

export class SQLishParser {
  parse(sql: string): Token[] {
    return parse(sql) as Token[];
  }
}
```

### 2.3 Replace `src/pegjs.d.ts` with `src/sqlish-grammar.d.ts`
```typescript
export function parse(input: string): unknown;
```

### 2.4 Create `tsup.config.ts`
Two entry points — `index` (core) and `react` (optional):
```typescript
import { defineConfig } from 'tsup';
export default defineConfig([
  {
    entry: { index: 'src/index.ts' },
    format: ['esm'],
    dts: true,
    outDir: 'dist',
    clean: true,
    sourcemap: true,
  },
  {
    entry: { react: 'src/react.tsx' },
    format: ['esm'],
    dts: true,
    outDir: 'dist',
    clean: false,
    sourcemap: true,
    external: ['react', 'react-dom'],
    esbuildOptions(options) { options.jsx = 'automatic'; },
  },
]);
```

Produces: `dist/index.js`, `dist/index.d.ts`, `dist/react.js`, `dist/react.d.ts`

## Phase 3: Package.json Overhaul

Full rewrite. Key points:
- `name`: `@sentry/sqlish`, `version`: `0.0.0` (Craft bumps on release)
- `"type": "module"` (ESM package)
- `exports` map with `.` and `./react` subpaths
- `files`: `["dist"]` — only ship built output
- `license`: `"Apache-2.0"`
- `publishConfig.access`: `"public"` (scoped packages need this)
- `peerDependencies`: `react >=17` (optional via `peerDependenciesMeta`)
- **Remove** from deps: `@sentry/react`, `react`, `react-dom` (all runtime deps gone)
- **Remove** from devDeps: `webpack`, `webpack-cli`, `ts-loader`, `@types/react-dom`, `jest`, `ts-jest`, `@types/jest`
- **Add** to devDeps: `tsup`, `vitest`
- Scripts: `"prebuild": "node scripts/compile-grammar.mjs"`, `"build": "tsup"`, `"test": "vitest run"`

## Phase 4: Craft + CI/CD

### 4.1 Create `.craft.yml`
```yaml
minVersion: '2.21.0'

changelog:
  policy: auto

targets:
  - name: npm
    access: public
  - name: github
```

### 4.2 Create `.github/workflows/ci.yml`
Triggers on: `push` to `main` and `release/**` branches, and `pull_request`.

Steps:
1. `actions/checkout@v4`
2. `pnpm/action-setup@v4` (auto-detects version from `packageManager` field)
3. `actions/setup-node@v4` with `node-version: 20`, `cache: pnpm`
4. `pnpm install --frozen-lockfile`
5. `pnpm run build`
6. `pnpm test`
7. `pnpm pack`
8. `actions/upload-artifact@v4` with name `${{ github.sha }}`, path `*.tgz`

The `release/**` branch trigger is **critical** — Craft creates release branches and CI must build artifacts on them.

### 4.3 Create `.github/workflows/release.yml`
```yaml
name: Release
on:
  workflow_dispatch:
jobs:
  release:
    uses: getsentry/craft/.github/workflows/release.yml@v2
    secrets: inherit
```

## Phase 5: Switch to Vitest

Delete `jest.config.js` entirely. Vitest needs zero config for this project — it natively handles TypeScript and ESM. The `vitest run` command discovers `*.test.ts` / `*.spec.ts` files automatically.

No `vitest.config.ts` needed unless we want custom settings later. Vitest picks up `tsconfig.json` for TS compilation automatically.

---

- Delete `package-lock.json`

- Run `pnpm install` to generate `pnpm-lock.yaml`
- Add `.npmrc` with `shamefully-hoist=false` (pnpm default, explicit for clarity — optional)
